Proximal Policy Optimization
PPO constrains policy updates using a clipped surrogate objective, preventing large destructive updates while maintaining sample efficiency.
L_CLIP(θ) = E_t [ min( r_t(θ)·A_t, clip(r_t(θ), 1-ε, 1+ε)·A_t ) ] r_t(θ) = π_θ(a_t|s_t) / π_θ_old(a_t|s_t)
Clip Epsilon (ε)
ε = 0.20
Advantage Sign
Key Innovation

The clipping prevents the policy ratio from moving too far from 1, acting as an implicit trust region without the computational overhead of TRPO's constraint optimization.

In Agent Lightning

PPO is the default optimizer for RLHF pipelines within Agent Lightning. The framework extends it with per-token advantage estimation for LLM action spaces.

Policy Gradient
GRPO — Group Relative Policy Optimization
GRPO eliminates the value network by computing advantages relative to a group of sampled responses. Click any sample to randomize its reward.
A_i = (r_i - mean(r)) / std(r) (group-normalized advantage) L_GRPO = E_i [ min( r_i(θ)·A_i, clip(r_i(θ),1-ε,1+ε)·A_i ) ] - β·KL(π_θ || π_ref)

Key idea: Sample G responses from the same prompt, compute rewards for all, normalize to get advantages. No critic network needed — cheaper than PPO for LLMs.

Preference-Based
RLHF — Reinforcement Learning from Human Feedback
A three-stage pipeline that trains a language model to align with human preferences using comparisons rather than scalar rewards.

Stage 1: Supervised Fine-Tuning (SFT)

Start with a pre-trained language model and fine-tune it on a curated dataset of high-quality demonstrations. This creates the initial policy π_SFT that can follow instructions.

  • Collect human-written demonstrations for target tasks
  • Standard cross-entropy training: L = -Σ log π(y_t | x, y_{<t})
  • Result: a capable but not yet preference-aligned model
L_SFT = -E_{(x,y)~D} [ Σ_t log π_θ(y_t | x, y_{<t}) ]

Stage 2: Reward Model Training

Train a separate reward model r_φ to predict human preferences. Given a prompt and two responses, humans label which they prefer. The reward model learns to score responses.

  • Collect pairwise preference data: (x, y_w, y_l) where y_w ≻ y_l
  • Bradley-Terry model: P(y_w ≻ y_l) = σ(r_φ(x,y_w) - r_φ(x,y_l))
  • Result: a scalar reward function aligned with human preferences
L_RM = -E [ log σ( r_φ(x, y_w) - r_φ(x, y_l) ) ]

Stage 3: RL Fine-Tuning with PPO + KL Penalty

Optimize the SFT policy using PPO to maximize the reward model signal, while adding a KL divergence penalty to prevent the policy from drifting too far from π_SFT.

  • Reward: r(x,y) = r_φ(x,y) - β · KL(π_θ || π_SFT)
  • KL penalty prevents reward hacking and maintains coherence
  • Result: a policy that scores high on the reward model while staying sensible
max_θ E_{x~D, y~π_θ} [ r_φ(x,y) - β · KL( π_θ(·|x) || π_SFT(·|x) ) ]
Preference-Based
DPO — Direct Preference Optimization
DPO bypasses the reward model entirely, directly optimizing the policy from preference data using an implicit reward derived from the policy ratio.
L_DPO(θ) = -E_{(x,y_w,y_l)} [ log σ( β · log(π_θ(y_w|x)/π_ref(y_w|x)) - β · log(π_θ(y_l|x)/π_ref(y_l|x)) ) ]
Beta (β) Parameter
β = 0.10

β controls the KL divergence from the reference policy. Higher β → stronger regularization, policy stays closer to reference. Lower β → more aggressive preference optimization.

Implicit reward: r*(x,y) = β · log(π_θ(y|x) / π_ref(y|x)) — the log-ratio between trained and reference policy is itself the optimal reward function.

Async Design

Multiple clients run rollouts concurrently. The server updates the policy whenever a batch is ready, without blocking clients.

Any Agent Type

Clients can run single-LLM, tool-augmented, ReAct, multi-agent, or hierarchical agents — the protocol is agent-agnostic.

Scalability

Scale to hundreds of parallel clients. The server handles batching, prioritized replay, and gradient accumulation automatically.

Masking vs. Decomposition
Two complementary strategies for handling mixed-content trajectories in LLM agents: token masking and reward decomposition.
Token Masking

Selectively include/exclude token positions from the policy gradient loss. Tool-call tokens, system prompts, and retrieved context are masked out — only agent decision tokens contribute to learning.

[SYS]Youareahelpfulassistant Think:Ishouldsearchforthis [TOOLRESULT]Parisisthecapital Answer:ThecapitalisParis.
A Agent tokens (trained) A Masked tokens (ignored) A Final answer (reward signal)
Reward Decomposition

Instead of a single terminal reward, decompose the trajectory reward into step-wise components. Each intermediate action receives a partial reward signal based on its contribution to the final outcome.

r_total = Σ_i w_i · r_i r_i ∈ { r_plan, r_tool_select, r_tool_exec, r_synthesis, r_final }
