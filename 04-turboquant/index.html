The Memory Crisis
Every time an AI reads a long document, it stores a massive table of numbers called a KV cache. At 128,000 words, that cache uses 4GB of expensive GPU memory — per model layer. TurboQuant squeezes it down to 1GB with zero quality loss.

AI models store millions of numbers as "memory" when reading long documents. These numbers need to be compressed 4× to fit on practical hardware. What Is a Vector? (and why compress it?)

An attention key vector is just a list of numbers — one per dimension of the model. For Llama-3.1-8B that's 4,096 numbers per token. Quantization replaces each 16-bit float with a 3–4 bit integer — losing a tiny bit of precision but saving 4–5× memory.

Method — Step 1 of 3
Step 1 — Scramble the Vector
Real embedding vectors cluster and spike — a few coordinates dominate, making naive compression wasteful. A random rotation spreads energy evenly across all coordinates, making each one equally compressible.
The core idea

Before rotation, a few coordinates carry most of the information. Compressing equally is wasteful — some bins cover noise, others cover crucial signal. After a random rotation matrix R, every coordinate carries exactly ‖x‖²/d energy. They all become equally important, and equally easy to compress.

Everyday analogy

Rotating the data is like shuffling a deck of cards before compression — it distributes information evenly so no single card carries too much importance. After shuffling, every card is equally likely to be "important," so you can compress all cards with the same quality.

R ~ Haar(O(d)) — uniform random orthogonal matrix E[‖(Rx)_i‖²] = ‖x‖²/d for all i Energy is exactly equalized across all d coordinates. Each rotated coordinate (Rx)_i has the same expected squared magnitude.
Equal energy per coordinate gives every coordinate the same statistical shape. Step 2 shows what that shape is — and why it's a gift. Step 2: Beta Distribution →
Method — Step 2 of 3
Step 2 — The Beta Distribution
After rotation, each coordinate follows a Beta distribution. For large dimensions, this concentrates tightly — making all coordinates nearly identical in variance. That's the key to treating them independently and designing an optimal quantizer.
Why the Beta distribution matters

If x is a unit vector in d dimensions, then after a random rotation, the squared value of each coordinate follows Beta(1/2, (d−1)/2). As d grows large, this distribution concentrates near 1/d — meaning every coordinate has almost exactly the same variance. This is what allows us to quantize each coordinate independently with the same codebook.

Everyday analogy

Correlations between coordinates are like knowing someone's left shoe size tells you their right. Once you factor that out (rotate), you can compress each dimension independently without losing accuracy. The Beta distribution is nature telling you exactly how spread out each coordinate is — so you can design a perfect compression bin for it.

Known distribution + independent coordinates = we can now design the mathematically optimal quantizer for each coordinate. That's Step 3. Step 3: Quantization →
Method — Step 3 of 3
Step 3 — Optimal Scalar Quantization
Lloyd-Max quantization finds the optimal bin boundaries for a known distribution. With the Beta distribution locked in from Step 2, each coordinate gets a custom codebook — and TurboQuant stays within 2.14× of the information-theoretic limit.
The Lloyd-Max algorithm

Given a known distribution, Lloyd-Max finds the quantization thresholds that minimize mean squared error. The rule: each threshold lies halfway between adjacent centroids, and each centroid is the mean of its bin. Repeat until convergence. For a Gaussian, this puts bins densely in the center and sparsely at the tails.

The Hidden Trap
The Bias Problem & Fix
Minimizing MSE and preserving inner products are different objectives. MSE-optimal quantization systematically overestimates inner products. TurboQuant_PROD fixes this with a 1-bit correction, achieving unbiased estimates at negligible extra cost.
Why this matters for attention

The core operation in attention is the inner product ⟨query, key⟩. If your quantization systematically overestimates these products, the attention scores will be biased — some tokens will seem more relevant than they are. TurboQuant_PROD adds a single extra bit per vector that mathematically removes this bias.

Open Problems
Adaptive bit allocation per dimension▼
TurboQuant uses the same number of bits for every coordinate. Dimensions that carry more information (even after rotation) could benefit from more bits. Optimal adaptive allocation would require solving a constrained optimization problem per vector — potentially bringing distortion even closer to the Shannon bound.
Non-uniform random rotations tuned to data▼
The Haar-distribution rotation is data-agnostic. If you know the covariance structure of your embeddings, a PCA-whitening rotation might do better. The tradeoff: data-dependent rotations require an offline training phase, losing TurboQuant's online guarantee.
Heavy-tailed embedding distributions▼
The Beta distribution result assumes the input vector lies on a unit sphere. Real embeddings may have heavy tails or non-unit norms. Understanding how robustly TurboQuant's guarantees hold under these violations — and how to normalize in practice — is an open question.
Integration with quantization-aware training▼
TurboQuant operates post-hoc on frozen model weights. Training models with TurboQuant-style quantization in the loop could allow the model to learn representations that are even easier to compress — potentially pushing below 3 bits while maintaining quality.
